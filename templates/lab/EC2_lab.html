EC2 Instance Launch Lab

Learn how to launch and configure an EC2 instance with proper security settings.

Objective

By the end of this lab, you will be able to launch an EC2 instance in a specified region, configure a security group to allow inbound SSH access, assign the correct IAM role, and verify that the instance is running correctly. This is a fundamental skill for cloud computing on AWS.

Rules

  • Use only the provided environment and credentials.
  • Do not create any extra resources not specified in the instructions.
  • The lab environment will automatically expire after the time limit (60 minutes).

Instructions

1

Launch EC2 Instance

▼

Navigate to the EC2 service in the AWS Console. Click 'Launch instances' and enter a name. Select 'Amazon Linux 2' for the AMI and set the instance type to t2.micro. Ensure the default VPC and a public subnet are selected.

Remember to select the specific key pair provided in the 'Connect to your environment' section.

2

Configure Security Group

▼

In the Network settings section during the instance launch process, create a new security group. Add a single inbound rule to allow SSH traffic (port 22) with the source set to 'My IP'. Give the security group a descriptive name like 'ssh-access'.

3

Attach IAM Role

▼

Expand the 'Advanced details' section. In the 'IAM instance profile' dropdown, select the IAM role provided for this lab. This role has the necessary permissions for the instance to communicate with other services.

4

Verify Instance

▼

Click 'Launch instance' to start the process. Once the instance creation is initiated, navigate back to the EC2 instances list. Wait a few moments and ensure that your new instance's status is running.
