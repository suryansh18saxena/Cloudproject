EC2 Instance Launch Lab

Learn how to launch and configure an EC2 instance with proper security settings on AWS.

flag Objective

By the end of this lab, you will be able to launch an EC2 instance in a specified region, configure a security group to allow inbound SSH access, and verify that the instance is running correctly.

gavel Rules
  • Use only the provided environment and credentials.
  • Do not create any extra resources not specified in the instructions.
  • The lab environment will automatically expire after 60 minutes.
checklist Instructions
1

Launch EC2 Instance

expand_more

Navigate to the EC2 service in the AWS Console. Click 'Launch instances' and enter a name. Select 'Amazon Linux 2' for the AMI and set the instance type to t2.micro. Ensure the default VPC and a public subnet are selected.

💡 Remember to select the specific key pair provided.

2

Configure Security Group

expand_more

In the Network settings section, create a new security group. Add a single inbound rule to allow SSH traffic (port 22) with the source set to 'My IP'. Give it a descriptive name like 'ssh-access'.

3

Attach IAM Role

expand_more

Expand the 'Advanced details' section. In the 'IAM instance profile' dropdown, select the IAM role provided for this lab.

4

Verify Instance

expand_more

Click 'Launch instance'. Navigate to the EC2 instances list and wait for your instance status to become running.
