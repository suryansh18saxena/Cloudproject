S3 Architecture Lab

Learn how to securely configure an S3 bucket with versioning, policies, and block public access.

menu_book 📖 Read Study Material
flag Objective

By the end of this lab, you will be able to launch an S3 bucket, enable versioning, configure bucket policies, block public access, and upload objects successfully.

gavel Rules
  • Use only the provided environment and credentials.
  • Do not create any extra resources not specified in the instructions.
  • The lab environment will automatically expire after 60 minutes.
checklist Instructions
1

Create S3 Bucket

expand_more

Navigate to S3. Click Create Bucket. Name it securely. You must create it in the us-east-1 region.

2

Enable Versioning

expand_more

Go to the Properties tab of your bucket and Edit Bucket Versioning to Enable it.

3

Configure Policy & Block Public Access

expand_more

Ensure Block Public Access is turned ON. Then attach a basic bucket policy allowing your IAM user.

4

Upload an Object

expand_more

Upload any file to your newly created S3 bucket to test the configuration.
